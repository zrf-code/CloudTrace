# ☁️ CloudTrace 云迹

[![Python 3.6+](https://img.shields.io/badge/Python-3.6%2B-blue.svg)](https://www.python.org/)
[![License](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

**CloudTrace 云迹** 是一款带有现代化图形界面的 Cloudflare IP 扫描与测速工具。  
融合了 [XIU2/CloudflareSpeedTest](https://github.com/XIU2/CloudflareSpeedTest) 的高效测速逻辑与 [xiaolin-007/CloudFlareScan](https://github.com/xiaolin-007/CloudFlareScan) 的美观 UI 设计理念。

</div>

## ✨ 功能特性

### 🚀 智能高效扫描
- **全异步高并发**：基于 `asyncio` + `aiohttp` 实现，支持自定义并发数（最高 500+），极速完成海量 IP 筛选。
- **智能随机采样**：从 Cloudflare 官方 CIDR 段智能拆分 /24 子网并随机生成 IP，避免无意义遍历，大幅提高优质 IP 命中率。
- **精准 TCP 延迟**：采用 TCP Ping 替代传统 ICMP，有效绕过禁 Ping 环境，多次测试取最小值，数据更真实可靠。

### ⚡ 专业下载测速
- **内置真实测速**：通过 `speed.cloudflare.com` 下载大文件实测带宽，排除 ICMP 延迟与实际下载速度不符的干扰。
- **地区专属测速**：支持输入 IATA 代码（如 HKG、LAX），单独对特定地区的节点进行深度测速，精准锁定最优区域。
- **灵活参数控制**：自定义测速数量、延迟阈值、测速端口（支持 443, 2053, 2083 等），满足不同网络环境需求。

### 🎯 精准节点识别
- **双栈全面覆盖**：全面支持 Cloudflare 官方 IPv4 与 IPv6 地址段扫描。
- **IATA 深度解析**：自动获取节点的 IATA 机场代码，并内置 80+ Cloudflare常见代码的中文汉化映射（如 NRT->东京, HKG->香港）。
- **双重识别机制**：优先解析 `/cdn-cgi/trace`，若失败自动回退解析 HTTP 响应头中的 `CF-RAY` 字段，确保识别成功率。

### 🎨 现代化交互体验
- **精美深色 UI**：采用深蓝色渐变主题设计，告别枯燥命令行，带来极致视觉享受。
- **丰富状态反馈**：测速表格自带红黄绿颜色预警（延迟/速度），前三名金银铜高亮；双击单元格即可快速复制 IP 或地区码。
- **全自定义弹窗**：摒弃原生系统对话框，使用统一风格的自定义消息框与确认弹窗，交互体验更优雅。

### 💾 完善的数据管理
- **自动历史存档**：扫描与测速结果自动带时间戳保存，并实时更新 `latest` 副本，支持一键加载历史记录，无需重复扫描。
- **智能空间清理**：自动清理旧版命名格式文件，自动保留最新 5 份历史记录，避免磁盘空间无序浪费。
- **多格式导出**：支持将结果一键导出为 CSV 或 JSON 格式，方便二次处理。

### ⚒️️ 极致系统兼容
- **Win7 深度适配**：底层强制使用 TLS 1.2 协议并切换事件循环策略，完美解决 Windows 7 下 PySide6 崩溃与 SSL 握手失败问题。

## 🚄 安装与运行

### 环境要求
- Python 3.6 及以上版本
- Windows 7 / 10 / 11 或其他主流操作系统

### 安装步骤

1. 克隆本项目：
   ```bash
   git clone https://github.com/zrf-code/CloudTrace.git
   cd CloudTrace
   ```

2. 安装依赖：
   ```bash
   pip install -r requirements.txt
   ```

3. 运行程序：
   ```bash
   python CloudTrace.py
   ```

## 📖 使用说明

### 只需简单三步即可找到最快的 Cloudflare IP：

1. **获取可用 IP**：点击「IPv4 扫描」或「IPv6 扫描」，等待进度条跑完。
2. **测试下载速度**：扫描完成后，输入地区码点击「地区测速」，或直接点击「完全测速」。
3. **复制与导出**：在下方测速结果表格中双击 IP 即可复制，或点击「导出结果」保存为文件。

---

### 📋 核心操作详解

#### 1. 扫描与加载
- **首次扫描**：选择 IPv4 或 IPv6 开始扫描。程序会并发测试海量 IP 的延迟，并自动剔除超时及高延迟的 IP。
- **加载历史**：如果之前已经扫描过，可以直接点击「加载 IPv4/IPv6 扫描结果」，无需重新扫描即可直接进行测速。

#### 2. 测速模式
测速必须基于已有的扫描结果（或加载的历史结果）：
- **地区测速**：在「地区码」输入框填写 3 位 IATA 代码（如 `HKG`、`NRT`、`LAX`），程序会筛选出属于该地区的 IP 进行下载测速。
  > 💡 **提示**：可在扫描状态和统计信息中查看当前可用的地区码列表。输入地区码时会自动转为大写。
- **完全测速**：不填地区码，直接点击「完全测速」。程序会自动选取延迟最低的前 N 个 IP（N 由「数量」参数决定）进行下载测速。

#### 3. 结果处理
- **快速复制**：在测速结果表格中，**双击任意单元格**（如 IP 地址、地区、延迟等），即可将该内容快速复制到剪贴板，方便直接粘贴到其他工具中使用。
- **导出文件**：点击「导出结果」，可将扫描和测速结果分别导出。支持 `CSV` 格式（可用 Excel 打开）和 `JSON` 格式。

### ⚙️ 参数配置指南

在主界面的参数面板中，您可以根据自己的网络情况调整以下参数：

| 参数 | 说明 | 调优建议 |
| :--- | :--- | :--- |
| **地区码** | Cloudflare 节点 3 位代码 | 想用哪个地区的节点就填哪个。常见：`HKG`(香港)、`TPE`(台北)、`NRT`(东京)、`SIN`(新加坡)、`LAX`(洛杉矶) |
| **数量** | 参与下载测速的 IP 数量 | 默认 10。如果不是专门做对比测试，建议设为 5-10，因为测速较耗时，数量过多等待时间较长。 |
| **端口** | 测试连接的目标端口 | 默认 443。如果 443 端口被干扰或封禁，可尝试切换为 `2053`、`2083` 等其他支持 TLS 的端口。 |
| **并发** | 扫描阶段同时测试的连接数 | 默认 200。根据自己的网络环境调整，宽带较慢或路由器性能较弱时，建议调低至 50-100 以防断网 |
| **阈值** | 延迟过滤阈值 | 默认 230。只保留延迟低于此数值的 IP。国内一般建议 200 以内，若只要极低延迟节点可设为 150 甚至 100。 |

> ⚠️ **注意**：修改端口、并发、阈值等参数后，需**重新进行扫描**，新参数才会在扫描阶段生效。

## ⚙️ Win7 兼容说明

在 Windows 7 环境下运行 PySide6 与异步网络程序通常会遇到 SSL 握手失败或事件循环崩溃的问题。本项目已在底层进行了适配：
- 强制使用 `WindowsSelectorEventLoopPolicy` 替代默认的 Proactor 策略。
- 针对 Win7 的 SSL 上下文进行了修补，强制使用 TLS 1.2 协议。
- 关闭了 Qt 的高 DPI 缩放，避免在 Win7 上出现界面渲染错乱。
